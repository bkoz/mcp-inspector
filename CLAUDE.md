## Project Overview

This repository contains a mini tutorial on using the MCP Inspector. It demonstrates how to build a basic MCP (Model Context Protocol) server that can be inspected and tested using MCP Inspector tools.

## Architecture

### MCP Server Implementation

The project uses the `@modelcontextprotocol/sdk` to create an MCP server that communicates via stdio transport. The server architecture follows this pattern:

1. **Server Initialization** (src/index.ts): Creates a Server instance with metadata (name, version) and declares capabilities (tools support)

2. **Tool Definition**: Tools are registered using `ListToolsRequestSchema` handler. Each tool must specify:
   - `name`: Unique identifier
   - `description`: What the tool does
   - `inputSchema`: JSON Schema defining the tool's parameters

3. **Tool Execution**: Tool calls are handled via `CallToolRequestSchema` handler, which receives the tool name and arguments, executes the logic, and returns content (typically text responses)

4. **Transport Layer**: Uses `StdioServerTransport` for communication, allowing the server to be invoked as a subprocess and inspected by MCP Inspector or integrated into MCP clients

### Adding New Tools

To add a new tool:
1. Add the tool definition in the `ListToolsRequestSchema` handler
2. Implement the tool logic in the `CallToolRequestSchema` handler
3. Return results in the MCP content format (array of content blocks)
